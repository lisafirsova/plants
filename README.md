# Plants

Android-приложение для ведения домашнего сада на C# и .NET 10. Телефон обращается к защищенному REST API, а сервер хранит данные в PostgreSQL `plants_db` и выполняет запросы к OpenAI.

## Возможности

- Google-авторизация с сохранением пользователя в PostgreSQL.
- Карточки растений, категории, виды, фазы жизненного цикла, состояние и заметки.
- Календарь ухода в режимах день, неделя и месяц.
- Полив, подкормка, обрезка и пересадка с возможностью выполнить и отменить выполнение.
- Дата начала расписания может быть выбрана от трёх дней до текущей даты и далее.
- Локальные Android-уведомления по расписанию.
- Фотоархив с названием и датой каждого снимка.
- Поиск, фильтрация и сортировка растений, фото, вредителей и болезней.
- ИИ-чат с вопросами, историей диалога и анализом прикреплённых фотографий.
- Администрирование видов растений, справочника, средств защиты и алгоритмов рекомендаций.
- Экспорт пяти аналитических листов в Excel-совместимый файл.

## PostgreSQL

Прямое подключение Android-приложения к PostgreSQL не используется. Строка подключения задается только на сервере через `DATABASE_URL` или `PLANTS_DATABASE_CONNECTION`. Схема локальной базы:

```powershell
$env:PGPASSWORD="ВАШ_ПАРОЛЬ"
& "C:\Program Files\PostgreSQL\18\bin\psql.exe" `
  -h localhost -U postgres -d plants_db `
  -f ".\Database\schema_3nf_postgresql.sql"
```

При запуске приложение само выполняет безопасные повторяемые миграции.

## Настоящий ИИ

Ключ OpenAI нельзя помещать в Android-приложение. Он используется только сервером `Plants.AiProxy`.

1. Откройте `https://platform.openai.com/` и войдите в аккаунт.
2. Создайте или выберите проект в OpenAI Platform.
3. В разделе Billing добавьте способ оплаты и положительный баланс API. Подписка ChatGPT Plus не является балансом API.
4. Откройте API keys, нажмите `Create new secret key` и сразу сохраните полученный ключ.
5. В PowerShell перейдите в корень проекта и запустите прокси:

```powershell
cd "C:\Users\user\Documents\New project 2"
$env:OPENAI_API_KEY="sk-..."
$env:OPENAI_MODEL="gpt-5.4-mini"
dotnet run --project .\Backend\Plants.AiProxy\Plants.AiProxy.csproj
```

6. Не закрывайте это окно PowerShell: сервер должен работать одновременно с приложением.
7. В браузере компьютера откройте `http://localhost:5079/health`. Ответ должен быть `{"status":"ok"}`.
8. Запустите Android-эмулятор и приложение. Эмулятор обращается к компьютеру по `http://10.0.2.2:5079`.
9. Добавьте растение, затем откройте `Справочник → ИИ-консультант`.
10. Напишите вопрос или нажмите «Прикрепить фото». Сервер передаст карточки растений, историю чата и выбранное изображение в OpenAI.

Если чат пишет, что сервер недоступен, проверьте, что PowerShell с `dotnet run` всё ещё открыт. Если сервер возвращает ошибку `401`, создайте новый API key. Ошибки `429` обычно означают отсутствие доступного API-баланса или превышение лимита.

Для реального телефона замените `AiProxyConfig.BaseUrl` на HTTPS-адрес сервера или IP компьютера в локальной сети. В production используйте HTTPS и храните ключ в секретах сервера.

## Сборка

```powershell
dotnet workload restore
dotnet build Plants.csproj
```

В Visual Studio выберите Android-эмулятор как целевое устройство и нажмите зелёную кнопку запуска.
